Basic Concepts

Core ideas behind logistic regression, explained simply

01 — What is Logistic Regression?

Logistic Regression is a supervised machine learning algorithm used to predict the probability that an observation belongs to one of two categories. Despite its name, it solves classification problems — not regression ones.

Instead of fitting a straight line through data, it applies the sigmoid function to map any real-valued input to a probability between 0 and 1. That probability is then used to assign a class label.

03 — Binary Classification

0 Negative class

The absence of the condition. The event did not occur.

e.g. Not obese / No disease / Email is not spam

1 Positive class

The presence of the condition. The event did occur.

e.g. Obese / Has disease / Email is spam

Binary classification is the simplest form of classification — the model must decide between exactly two mutually exclusive outcomes. Every observation is assigned either 0 or 1 based on the estimated probability.

04 — Target Variable & Class Label

Target Variable (Y)

The outcome the model is trained to predict. In binary problems, it takes only the values 0 or 1.

e.g. Obesity: 1 = obese, 0 = not obese

Class Label

The actual category assigned to each training example. The model learns to distinguish these labels from the input features.

e.g. "Spam" or "Not Spam"

05 — Features / Predictors

Features (also called predictors or independent variables) are the input values used to make a prediction. Each feature provides information that helps the model distinguish between the two classes.

Age Income Purchase History Time on Site Clicks Email Opens Location Device Type

06 — Probability Estimation

P(Y=1)  =  1  /  (1  +  e−z)

P(Y=1) — probability of class 1 e — Euler's number z — β₀ + β₁x₁ + … + βₙxₙ

The sigmoid function takes the linear combination of features and squashes it into a value between 0 and 1. This output is interpreted as the probability that the observation belongs to class 1.

07 — Decision Boundary

The decision boundary is the line (or surface) that separates the two classes in feature space. All points on one side are classified as 1; all points on the other side as 0.

In a model with two features, the decision boundary is a straight line. With more features, it becomes a hyperplane — but the logic is the same: it is where the predicted probability equals 0.5.

08 — Classification Threshold

The threshold is the cut-off probability used to convert the model's output into a class label. The default is 0.5:

← Predict 0 (Negative) Predict 1 (Positive) →
  • P ≥ 0.5 → the model predicts class 1 (positive).
  • P < 0.5 → the model predicts class 0 (negative).
  • The threshold can be adjusted based on context — lowering it increases sensitivity.

09 — Real-World Example

A common application of logistic regression is email spam detection. Given features extracted from an email, the model estimates the probability that it is spam:

z = −4.2 + 1.8×ContainsPromo + 0.9×LinksCount − 1.1×KnownSender

For a given email, suppose z = 2.1:

P(Spam) = 1 / (1 + e−2.1) ≈ 0.891 → Class 1
